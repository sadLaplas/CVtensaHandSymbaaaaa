# Stage 02 — Landmark dataset capture

## Goal of this stage

Цель этапа 02 — превратить базовый hand tracking из этапа 01 в практический инструмент сбора датасета. На этом шаге проект начинает не просто видеть руку, а сохранять подготовленные landmarks в воспроизводимом и удобном для обучения формате.

## What was added

- конфигурация блока `dataset` в `configs/config.yaml`;
- модуль `core/landmark_utils.py` для нормализации landmarks и построения feature vector;
- модуль `core/dataset_collector.py` для записи JSON-sample и индексного CSV;
- UI-контролы для выбора label и запуска/остановки записи;
- отдельная панель статуса записи в Streamlit;
- тесты для конфига, нормализации и записи датасета.

## What was changed

- `core/config.py` расширен dataclass-конфигом `DatasetConfig` и совместим с конфигами этапа 01;
- `ui/streamlit_app.py` теперь умеет записывать samples внутри существующего видеопотока;
- `README.md` обновлен под новый этап проекта;
- `.gitignore` дополнен правилами для IDE и автоматически записываемых raw-файлов.

## Why this stage matters

Распознавание жестов невозможно без качественного датасета. Этот этап создает мост между детекцией руки и будущим обучением модели:

- landmarks приводятся к стабильному виду;
- структура хранения становится предсказуемой;
- ручной сбор датасета в реальном времени становится частью основного UI;
- следующие этапы обучения и инференса смогут использовать уже подготовленный `feature_vector`.

## Architecture decisions

- Streamlit-интерфейс сохранен как текущий stable baseline. Миграция на Tkinter на этом этапе была бы ломающим изменением и не соответствовала бы накопительной архитектуре проекта.
- Нормализация landmarks вынесена в отдельный модуль `core/landmark_utils.py`, потому что эта логика понадобится и для training, и для inference.
- Запись датасета вынесена в `core/dataset_collector.py`, чтобы UI не занимался файловой системой напрямую.
- CSV-индекс хранится отдельно от JSON-sample: JSON удобен для отладки и повторного анализа, CSV — для быстрой выборки и подготовки train/validation split на следующих этапах.

## Detailed file-by-file overview

### `configs/config.yaml`

Модуль отвечает за:

- подготовку каталогов хранения;

Существующий live-preview расширен без смены архитектуры:

- запись выполняется прямо во время обработки кадров;

Проверяет создание JSON-sample, индексного CSV и защиту от неизвестных label.

## Important implementation details

### Landmark representation

Каждая рука представляется как:

- `original_landmarks`: список длиной `21`, где каждый элемент — `[x, y, z]`;
- `normalized_landmarks`: те же `21` точки после переноса в систему координат запястья и масштабирования;
- `feature_vector`: плоский список длиной `63`.

Это дает следующий ожидаемый shape:

- до flatten: `(21, 3)`
- после flatten: `(63,)`

### Why wrist-relative normalization was chosen

Подход с переносом начала координат в wrist-point дает инвариантность к положению руки в кадре. Дополнительное масштабирование по максимально удаленной точке снижает влияние дистанции до камеры. Такой формат удобен для будущей MLP-модели и дешево вычисляется на CPU.

### Dataset saving logic

Sample сохраняется только если:

- запись включена пользователем;
- рука найдена;
- если `require_single_hand=true`, в кадре ровно одна рука;
- выдержан `capture_interval_ms`;
- не достигнут `max_samples_per_label`.

### Performance considerations

- нормализация landmarks дешевая и выполняется только для одной выбранной руки;
- запись файла производится только при прохождении capture-условий, а не на каждом кадре;
- collector переиспользуется через состояние Streamlit, чтобы не пересканировать директории каждый кадр;
- CSV-индекс дописывается инкрементально без полной перезаписи.

## How to run

```bash
python3.11 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
streamlit run ui/streamlit_app.py
```

После запуска:

1. нажмите `Запустить обработку`;
2. выберите `Gesture label`;
3. нажмите `Включить запись`;
4. собирайте samples для каждого жеста.

## How to test

Проверка тестов:

```bash
python -m unittest discover -s tests -t .
```

Проверка синтаксиса:

```bash
python -m compileall .
```

Практическая проверка UI:

1. убедитесь, что `models/hand_landmarker.task` существует;
2. запустите Streamlit;
3. запишите несколько samples для `open_palm`;
4. проверьте появление файлов в `data/raw/open_palm/`;
5. проверьте обновление `data/raw/dataset_index.csv`.

## Known limitations

- пока сохраняются только landmarks, без кадров изображения;
- жесты еще не классифицируются;
- баланс классов контролируется пользователем вручную;
- текущий UI — Streamlit baseline проекта, а не отдельная desktop GUI;
- многорукие сценарии для записи намеренно ограничены ради качества датасета.
